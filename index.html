客戶需求分析

🎯 核心訴求

「我不希望限制系統只能在 591 或特定社團運作。
我希望這套 AI 就像網路上看到的 Clawbot 範例一樣——
當我給出一個『開發潛在售屋客戶』的目標時，
她能自己思考去哪裡找資料、自己判斷哪些是高價值目標，
並主動執行後續的觸及動作。」

4 大核心需求

1️⃣ 自主探索（Autonomous Exploration）

  • 不設限監控範圍
  • AI 自己決定去哪找客戶
  • 自主判斷平台價值
  • 動態調整搜尋策略
範例：
系統自己搜尋「竹山 賣屋」→ 發現 Facebook 社團 → 自動加入監控

2️⃣ 智能判斷（AI Reasoning）

  • 自己判斷高價值目標
  • 區分急售 vs 觀望
  • 評估客戶意願強度
  • 預測成交機率
範例：
看到「急售透天」→ 判斷：高價值（85 分）→ 優先觸及

3️⃣ 自動對話（Auto Engagement）

  • 主動發送開發訊息
  • 自動回覆客戶問題
  • 維持多輪對話
  • 判斷最佳切入點
範例：
發現賣屋貼文 → 自動留言「您好，請問這間房子還在嗎？」

4️⃣ 成交閉環（Sales Funnel）

  • 從發現到成交全流程
  • 自動篩選有意願名單
  • 建立 CRM 系統
  • 提醒跟進時機
範例：
整理高意願客戶 → Telegram 通知「張小姐該跟進了」

💡 關鍵差異

❌ 不是「自動化腳本」（固定流程、寫死邏輯）
✅ 是「AI 業務員」（自己思考、自己執行、自己學習）

傳統自動化：
• 單一腳本執行固定流程
• 無法並行處理
• 卡在一個環節就全停

• 4 個 AI 分工合作
• 並行處理、互不影響
• 像真人團隊一樣運作

實現方式詳解

需求 1：自主探索 → 如何用 OpenClaw 實現

使用工具： web_search + agent-browser + sessions_spawn
# 探索者 Sub-agent 工作流程 1. Main Session 下指令： sessions_spawn(task="去網路上找竹山賣屋客戶") 2. Sub-agent 自己執行： → web_search("竹山 賣屋 Facebook") → 找到 5 個相關社團 → agent-browser 訪問每個社團頁面 → 評估：成員數、活躍度、相關性 → 決定加入 2 個高價值社團 → gog sheets 更新監控清單 3. 回報 Main Session： 「今天新增 2 個平台，預估每日 10 個潛在客戶」
關鍵：不是寫死「去 Facebook 社團 A」，而是 AI 自己搜尋、自己判斷、自己決定

需求 2：智能判斷 → 如何用 OpenClaw 實現

使用工具： Claude Opus + subagents + memory
# 監控者 Sub-agent 工作流程 1. agent-browser 爬取社團貼文 → 每個貼文提取：標題、內容、聯絡方式、圖片 2. 呼叫 Claude Opus 判斷： Prompt: "這個賣屋貼文，請評估： - 是急售還是觀望？ - 屋主自售還是代理？ - 成交意願強度？ - 綜合評分 0-100" 3. Claude Opus 回答： { "type": "急售", "owner": true, "willingness": 90, "score": 85, "reason": "標題含『急售』、聯絡方式是手機、價格合理" } 4. 篩選：只保留評分 > 70 的客戶
關鍵：用 LLM 深度理解語意，而非簡單的關鍵字比對
