## 五、風險與注意事項

| 風險 | 影響 | 應對方式 |
|------|------|---------|
| 591 大改版 | 爬蟲需重寫（2-3 天工） | 列入維護合約，或額外報價 |
| LINE 龍蝦對話品質不佳 | 客戶不滿意 | 持續調 Prompt，收集真實對話優化 |
| FB 社團發文被刪 | 觸發反垃圾 | 降頻率 + 草稿審核模式 |
| Claude API 漲價 | 客戶月費增加 | 提前告知，可切換更便宜的模型 |
| 客戶要求超出範圍 | 免費做 = 虧 | 合約明確寫清楚交付範圍 |
| 開發時間超估 | 利潤被壓縮 | LINE 龍蝦是最大風險，預留 buffer |

### 建議合約重點

1. **交付範圍明確**：列出 12 個模組，超出範圍另行報價
2. **驗收標準**：按驗收 checklist 逐項確認
3. **API 費用歸屬**：合約明寫客戶自行負擔
4. **維護期**：驗收後 14 天（標準）/ 2 個月（專業）為免費維護期，之後需簽維護合約
5. **智財權**：程式碼交付後歸客戶所有，但框架/工具庫歸我方
6. **保密條款**：客戶資料不外洩，我方技術不揭露
